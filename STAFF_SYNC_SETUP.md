# Синхронизация на Staff от Discord

Стафът на сайта може да се попълва автоматично от Discord ролите. Всеки член с някоя от изброените по-долу роли се показва в секцията „Нашият Staff“ с **най-високата си роля** и снимка от Discord профила.

## Проверка на стаф роля (за достъп до сайта / админ панел)

За показване на „Админ панел“ и „Наръчник Полиция“ в навбара се използва **check-site-role**. Функцията определя ролята така:
- **Гражданин** – няма нито една от ролите по-долу.
- **Staff** – има поне една от: **SUPPORT TEAM**, **STAFF**, **STAFF LEADER** → админ панел само преглед.
- **Administrator** – има поне една от: **ADMINISTRATOR**, **DEVELOPER**, **PANEL ENGINEER**, **OWNER** (или остаряло име **CEO**, ако още съществува в сървъра) → пълен достъп.

Ролите се **проверяват по име** от Discord (`GET /guilds/{id}/roles`, case-insensitive). Ако няма съвпадение по име, се ползват резервните ID от таблицата по-долу.

Ако проверката не работи:
1. Увери се, че **DISCORD_BOT_TOKEN** е зададен в Supabase → Edge Functions → Secrets.
2. Ботът да е в същия сървър (guild). Използва се **DISCORD_GUILD_ID** или **DISCORD_POLICE_GUILD_ID**.
3. Деплой: `npx supabase functions deploy check-site-role`
4. Влез с **Discord** в админ панела – ролите се четат само при Discord вход.
5. Имената на ролите в Discord трябва да съвпадат с колоната „Роля“ (регистърът не влияе).

## Роли (от най-ниска към най-висока)

При синхронизация функцията **проверява ролите** по следния начин:
1. Взима списъка с роли на сървъра от Discord API (`GET /guilds/{id}/roles`).
2. Съпоставя по **име на ролята** (case-insensitive) с изброените по-долу етикети.
3. Ако в сървъра има роля с съответното име, използва се нейното ID от Discord; иначе се ползва резервното (хардкоднато) ID по-долу.

Така синхронизацията работи коректно и при различен сървър (различни ID-та), стига имената на ролите да съвпадат.

| Роля           | Резервно ID (fallback)   | Емотиконка |
|----------------|--------------------------|------------|
| SUPPORT TEAM | 1471238721549504763      | 🎫         |
| STAFF      | 1471238721549504765      | 🛡️         |
| STAFF LEADER  | 1471238721549504766      | ⚙️         |
| ADMINISTRATOR| 1471238721566146791      | 📝         |
| DEVELOPER   | 1471238721566146793      | 👥         |
| PANEL ENGINEER     | 1486054158753726637      | 📊         |
| OWNER       | 1471238721566146796      | 🚀         |


Добавят се **само** членове, които имат поне една от тези роли в Discord; за всеки се записва **най-високата** от тях.

### Override: `DISCORD_STAFF_ROLE_IDS`

Ако ролите в Discord са нови и **имената не съвпадат 1:1** с етикетите в таблицата, задай в Supabase → Edge Functions → Secrets:

`DISCORD_STAFF_ROLE_IDS` = **точно 7** ID-та в същия ред като таблицата (SUPPORT TEAM → … → OWNER), разделени със **запетая**, без интервали излишни.

Тогава синхронизацията **не** ползва съпоставяне по име за тези слотове.

### Поправка: всички членове на сървъра

По-стара версия на функцията е имала грешка в цикъла за пагинация на `GET /guilds/.../members` (след първите 1000 члена спираше). Актуалният `sync-staff-from-discord/index.ts` дърпа всички страници. **Редеплой** след промяната.

### Discord: Server Members Intent (задължително за списъка с членове)

За `GET /guilds/{id}/members` Discord изисква **Privileged Gateway Intent** **Server Members Intent** за бота:

1. [Discord Developer Portal](https://discord.com/developers/applications) → твоето приложение → **Bot** → **Privileged Gateway Intents** → включи **Server Members Intent** и запази.
2. **Ре-инвайтни** бота в сървъра (или премахни и добави отново), за да влязат в сила правата, ако Discord го изисква.
3. Ботът трябва да е **в същия guild** и да има права да вижда членове.

При **403** от Discord при синхронизация админ панелът показва подсказка; провери и **валидността на `DISCORD_BOT_TOKEN`** в Secrets при **401**.

## Деплой на функцията

**Не бъркай SQL Editor с Edge Functions:** файлът `SYNC_FUNCTION_PASTE_IN_SUPABASE.ts` е **TypeScript (Deno)** за функцията `sync-staff-from-discord`. Ако го пуснеш в **SQL Editor**, PostgreSQL ще гърми с `syntax error at or near "//"`. За базата ползвай само файлове `*.sql` от `supabase/` (напр. миграции, `RUN_THIS_*.sql`).

1. Миграция за новите колони (ако още не е пусната):
   ```bash
   npx supabase db push
   ```
   или прилагане на `supabase/migrations/20260307120000_staff_discord_sync.sql`.

2. Секрети (вече имаш `DISCORD_BOT_TOKEN` за другите функции):
   - Опционално за ежедневен cron: генерирай таен ключ и го задай като `CRON_SECRET` в Supabase → Edge Functions → Secrets.

3. Деплой на Edge Function:
   ```powershell
   Set-Location -LiteralPath "C:\Users\User\Desktop\chillrp\[Website]"
   npx supabase functions deploy sync-staff-from-discord
   ```

## Ръчна синхронизация

- В **Админ панел** → таб **Стаф** → бутон **„Синхронизирай от Discord“**.  
- Може да го ползват само потребители с поне една от стаф ролите от таблицата (напр. **OWNER**, **ADMINISTRATOR**, **STAFF** и т.н.).  
- Синхронизацията изтрива текущите записи със `source = 'discord_sync'` и ги презаписва от Discord (ръчно добавени стаф членове с `source = 'manual'` **не** се пипат).

## Ежедневно автоматично обновяване (cron)

За да се пуска синхронизацията всеки ден автоматично:

### Вариант A: Външен cron (cron-job.org, Uptime Robot и т.н.)

1. Създай таен ключ (например случайна дума) и го задай в Supabase:
   ```bash
   npx supabase secrets set CRON_SECRET="твоят_таен_ключ"
   ```

2. В [cron-job.org](https://cron-job.org) (или друг сервис):
   - **URL:** `https://zbvqakalrxaxkwbpmjhn.supabase.co/functions/v1/sync-staff-from-discord`
   - **Метод:** POST
   - **Header:** `X-Cron-Secret: твоят_таен_ключ`
   - **График:** ежедневно (напр. 04:00).

### Вариант B: Ръчно / при нужда

Ползвай бутона „Синхронизирай от Discord“ в Админ панела когато искаш да обновиш списъка.

---

## Важно

- Записът в базата използва **Discord име** (nick или global name), **аватар** от Discord и **емотиконката** за съответната роля.
- Ръчно добавени стаф членове (`source = 'manual'`) **не се изтриват** при синхронизация; презаписват се само записите от Discord (`source = 'discord_sync'`).
